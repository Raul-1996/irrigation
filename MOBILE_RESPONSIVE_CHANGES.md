# Мобильная адаптация status.html

## Выполненные изменения

### 1. Таблица зон → карточки на мобилке (<768px)

**HTML:**
- Добавлен контейнер `<div class="zones-cards" id="zones-cards">` для мобильных карточек
- Таблица `.zones-table` остаётся для десктопа

**CSS:**
- Новые классы: `.zone-card`, `.zone-card-header`, `.zone-card-body`, `.zone-card-actions`
- Карточки скрыты на десктопе, таблица скрыта на мобилке через `@media (max-width: 767px)`
- Цветовая индикация статуса через `border-left: 3px solid` (зелёный/серый)

**JavaScript:**
- Рендеринг **обеих** версий (таблица + карточки) в `loadZonesData()`
- Синхронное обновление данных (группы, следующий полив) для обоих представлений
- На мобилке в карточках время показывается компактно (HH:MM вместо полной даты)

### 2. Кнопки управления группой → вертикальный стек на мобилке

**CSS:**
```css
@media (max-width: 767px) {
  .btn-group {
    flex-direction: column;
    gap: 8px;
  }
  .btn-group button {
    width: 100% !important;
  }
}
```

- Кнопки "Остановить полив на 1/2/3 дня" теперь в столбик на мобилке
- Кнопки "Запустить/Остановить группу" — 100% ширины

### 3. Навбар — адаптация для мобилки

**base.html:**
```css
@media (max-width: 767px) {
  .logo-text { font-size: 1.1rem; }
  .logo-subtitle { font-size: 0.75rem; }
  nav a { padding: 0.4rem 0.6rem; font-size: 0.85rem; }
  main { padding: 1rem 0.5rem; }
}
```

- Уменьшены размеры шрифтов и отступы
- Пункты навигации компактнее, но остаются читаемыми
- Контент main-области — уменьшен padding для экономии места

### 4. Общие мобильные фиксы

- `overflow-x: hidden` на `body` — предотвращает горизонтальный скролл
- `box-sizing: border-box` уже был глобально (в base.html)
- Медиа-запрос изменён с `768px` на `767px` для чёткого разделения desktop/mobile
- Все контейнеры адаптивны, без фиксированных `min-width > 320px`

2. `test_mobile_buttons_responsive()` — проверка медиа-запроса `@media (max-width: 767px)` и `flex-direction: column`
