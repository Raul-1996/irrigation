## 0. Ответы на 3 вопроса Рауля (§1 задания)

### Вопрос 1 — точность Open-Meteo в регионе (особенно осадки)
- **Покрытие только глобальными моделями.** Для 77°E: ECMWF IFS (9 км), ICON Global (~13 км), GFS (~25 км). Региональные high-res (ICON-EU 23.5°W–62.5°E, ICON-D2, AROME) **не достают** — Киргизия за восточной границей доменов. Какую из трёх модель выбирает `best_match` — Open-Meteo не раскрывает (GH issue #214, #1690).
- **Температура — умеренно надёжна (±1-2°C).** Open-Meteo корректирует высоту через Copernicus DEM GLO-90 (90 м) + градиент 0.7°C/100 м. Для коэффициента ОК.
- **ET₀ — умеренно, возможна переоценка +15-25% летом.** ET₀ доминируется радиацией (60-80%), а NWP систематически завышают коротковолновую радиацию (bias ~+18 W/m², RMSE ~97). В ясную погоду (лето Иссык-Куля) ошибка меньше; при конвективной облачности — непредсказуема. Брать как **относительный индекс**, не абсолют.
- **Осадки — НЕнадёжны на суточном уровне в горах.** ERA5 wet bias 25-50%+ в аналогичных горах (Цилянь, Тибет). QPF FAR до 84% (H1). Годятся только для грубого «дождь/нет» на 1-2 дня. **Это фундамент всей логики локального гейтинга.**
- **Валидация возможна:** RP5.ru (архив Иссык-Куль аэропорт / Каракол, SYNOP 8×/сут) + Open-Meteo Previous Runs API (offset 1/3/7 дней) за 2022-2024. ~несколько часов Python. ERA5 как «истина» для осадков в горах НЕ годится (сам wet-biased).

Источники: open-meteo.com/en/docs/{ecmwf-api,historical-forecast-api,previous-runs-api}; ICON-EU domain (dwd-geoportal.de); ERA5 Qilian (mdpi.com/2073-4433/16/7/826); ERA5 Tibet wet bias (link.springer.com/article/10.1007/s00382-023-06672-3); rp5.ru/Weather_archive_in_Issyk-Kul.

### Вопрос 2 — дождь в мм, пороги
- **Interception (газон):** до **4.4 мм** дождя НЕ доходит до почвы (испаряется/перехватывается дёрном); дальше +16% каждого мм теряется. Кусты перехватывают 6-14%, деревья 10-40%.
- **Smart-контроллеры дефолт rain skip = 3.2 мм (1/8")** (Rachio, Hunter Rain-Clik, Rain Bird).
- **Текущий `rain_threshold_mm=5` — слишком низкий для ПОЛНОГО пропуска полива.** 5 мм реально даёт ~0.5 мм в почве = не замена поливу (норма полива газона 15-25 мм).
- **Рекомендованные пороги (мм/сутки):**
  | Диапазон | Действие | P_eff |
  |---|---|---|
  | < 3-4 мм | игнор | 0 |
  | 4-12 мм | кредит в баланс, полив НЕ пропускать | (P−4.4)×0.84 |
  | 12-25 мм | существенно, рассмотреть отмену | P×0.75-0.80 |
  | > 25 мм | полный пропуск | min(P×0.80, TAW) |
- Лучше: уйти от механического порога к **кредиту осадков в Dr**.
- Дневной USDA SCS / FAO формулы официально только месячные; для суток — практическое правило `P_eff = max(0, (P − 4.4)×0.8)`.

Источники: Turfgrass interception (pmc.ncbi.nlm.nih.gov/articles/PMC9439196, journals.plos.org/plosone PONE-0271236); FAO effective rainfall (fao.org/4/x5560e/x5560e03.htm); Rachio (support.rachio.com/.../rachio-weather-intelligence); Hunter Rain-Clik.

### Вопрос 3 — насколько сложной будет логика
- **Математика FAO-56 простая** — десятки строк, не сотни. Рекурсия `Dr += ETc − P_eff − I`, clamp `[0, TAW]`, полив при `Dr ≥ RAW`.
- **Минимально (v1):** ET₀ as-is × Kc(зона); TAW≈50-70 мм (газон/суглинок); p=0.4 (cool turf) / 0.5 (warm/декор); RAW=p×TAW; clamp; reset Dr→0 по сильному дождю; старт после дождя Dr=0.
- **Можно отложить:** dual Kc (Kcb+Ke); поправка ET₀ по VPD (псевдо-точность + риск двойного учёта — НЕ делать в v1); Hargreaves (только sanity-check); runoff/CR (обнулить); сезонные кривые Kc.
- **Главный риск — дрейф:** нет датчика влажности почвы → нечем замкнуть контур → накопленная ошибка. Митигации: reset по дождю + clamp + ночной rebalance по факту.
- **Честно:** точность ограничена двумя отсутствующими входами (дискретный дождь вместо мм; нет влагомера). Но это рабочая FAO-схема для планирования — ровно как Rachio/RainMachine/Netro для пользователей без датчиков. Выигрыш по воде у коммерции в поле 10-15% (не 40%). **Главная польза H2 — не экономия, а починка ночного занижения и скачков коэффициента.**

## 1. FAO-56 soil water balance — детали

- **TAW = 1000·(θ_FC − θ_WP)·Z_r** [мм]. Доступная влага по текстуре: супесь ~90 мм/м, суглинок ~130-150 мм/м, ил ~170 мм/м, глина ~120 мм/м (глина НЕ больше суглинка — высокий θ_WP). Z_r газон 0.5-1.0 м (практично 0.5).
  - Газон/суглинок: TAW = 1000·0.13·0.5 ≈ **65 мм** (диапазон 40-100 по Z_r). Супесь Z_r=0.4 → ~36 мм (лить чаще, меньше).
- **RAW = p·TAW.** p (Table 22): cool-season turf **0.40**, warm-season **0.50**, «многие культуры» 0.50. Поправка `p_adj = p + 0.04·(5−ETc)`, clamp 0.1-0.8. При ETc≈5.4 поправка ~0 — можно зашить константу.
- **Initial Dr:** датчика влажности нет → стартовать **после хорошего дождя/пролива → Dr=0**. Ошибка старта самозатухающая (первый же reset по дождю обнуляет). Вслепую в сухой период → Dr≈0.5·TAW.
- **Single Kc достаточно** (dual Kcb+Ke требует моделить испарение верхнего слоя — усложнение без выигрыша при дискретном датчике). Газон Kc≈0.9-0.95 практически постоянен.
- **Anti-drift обязательно:** (1) clamp Dr∈[0,TAW]; (2) deep percolation = нижний clamp Dr≥0 (вода сверх FC уходит, впрок не запасается!); (3) reset Dr→0 по сильному/долгому дождю. Опционально: периодический rebalance, учёт неполного смачивания. НЕ делать: Kalman/экспоненциальное забывание (нечем калибровать).
- **Runoff и капиллярный подъём (CR) — обнулить** (нет мм осадков; грунтовые воды глубоко).

Источники: FAO-56 Ch.8 (fao.org/4/x0490e/x0490e0e.htm, Eq.82-88, Table 19/22), Ch.6/7.

## 2. Crop coefficients Kc — рекомендуемые дефолты

| Тип зоны | Весна | Лето | Осень | Зима | Источник |
|---|---|---|---|---|---|
| Газон cool-season | 0.90 | 0.90 | 0.75 | 0.60 | UC ANR |
| Газон warm-season | 0.76 | 0.70 | 0.58 | 0.55 | UC ANR |
| Декор. кусты (умер.) | 0.50 | 0.50 | 0.40 | 0.20 | WUCOLS Ks=moderate |
| Деревья (листопадн.) | 0.45 | 0.75 | 0.55 | 0.25 | FAO-56 + UC ANR PF=0.5 |
| Огород (смешанн.) | 0.60 | 1.00 | 0.80 | — | FAO-56 усреднённый |
- **Дефолт неизвестного типа: Kc = 0.5** (UC ANR PF для любой смешанной зоны).
- **3 сезонные константы достаточно** (growth-stage tracking — только для огорода, разница Kc 0.5→1.1 = 2× полив).
- WUCOLS `KL = Ks×Kd×Kmc` практичнее single Kc для **смешанных** зон (учёт плотности+микроклимата). Для моно-газона — хватит одного Kc.

Источники: FAO-56 Ch.6 Table 12; UC ANR turfgrass-crop-coefficients; wucols.ucdavis.edu; cimis.water.ca.gov/Content/PDF/wucols00.pdf.

## 3. Anti-drift у коммерции (Rachio/RainMachine/Netro) — что применить

1. **Развести прогноз и факт (критично, урок Rachio pre-credit bug).** Прогнозный дождь → ТОЛЬКО решение «лить ли сегодня». В баланс Dr → ТОЛЬКО фактически выпавший дождь. Никогда не кредитовать прогнозом до факта (Rachio так недополивал осенью).
2. **Ночной rebalance по факту** (RainMachine Forecast Correction + Netro nightly) — **ядро замыкания контура без датчика почвы.** Ночью брать у Open-Meteo фактические/past-day ET₀ и осадки, перезаписывать Dr. Гасит 80-90% накопленной ошибки.
3. **Двусторонний clamp:** Dr≥0 при насыщении (Rachio cap 110% FC, RM «AW»-буфер по уклону), Dr≤TAW (не суше wilting point).
4. **Reset при сильном дожде:** факт ≥ Dr → Dr=0, полив сдвинуть.
5. **Якорь к климат-норме** (RainMachine: ETc/ETavg) — страховка от системной ошибки ET.
6. **Min/max ограничители** полива — backstop от ухода в крайность.
7. **Ручная корректировка Dr + логирование «прогноз vs факт»** для аудита дрейфа (коммерция в поле даёт 10-15%, требует ручной сверки).

Источники: community.rachio.com (pre-credit bug, soil-moisture-depletion); support.rainmachine.com (Forecast Correction, Field-Capacity); netrohome.com FAQ + патент US 11,860,652 (sensorless balance); RainMachine open-source = только ET₀ ASCE (github.com/sprinkler/rainmachine-developer-resources).

## 4. Локальный детект дождя по датчикам (идея Рауля) — ЧЕСТНАЯ оценка

- **Посылка верна:** точечный/сеточный прогноз плохо ловит локальную конвекцию (дождь в 3 км). Желание подтверждать локально — методологически здраво.
- **Механизм «скачок RH + падение T = дождь» реален** (wet-bulb evaporative cooling; gust front ΔT 1-3°C, >5°C за 5 мин не редкость). ML: влажность — главный предиктор дождя, ~80% точности по {RH, dewpoint, T, P} даже без датчика дождя.
- **Но по ОДНОЙ влажности — шумно:** туман (T−Tdew<2.5°C), роса (ночное радиационное охлаждение) дают RH 90-100% без дождя. Дискриминатор — **скорость и синхронность** (дождь = резкая ступенька; роса = плавно за часы). Важно: дождь бывает и при RH 60% → отсутствие скачка ≠ нет дождя.
- **Дискретный rain_sensor (bool):** прямее, но залипает мокрым после дождя (без подогрева — десятки минут), ложные от росы/полива, пропускает морось (задержка до ~103 с при 0.1 мм/ч). **Проверить, есть ли подогрев у датчика объекта.**
- **Sensor fusion (rain_sensor + ΔRH + ΔT + согласование с OM) — самый сильный вариант**, надёжность бинарного «был/не был» средняя→высокая. Truth table — две ценные диагонали:
  - OM обещал дождь + локально пусто (rain=false, нет скачка) → **обнулить осадки** (прогноз промахнулся);
  - OM молчал + rain_sensor=true (+скачок) → **засчитать факт** (локальный дождь).
- **Количество (мм) — НЕЛЬЗЯ** ни по длительности is_rain, ни по амплитуде RH. Даже капле-считающие пьезо-сенсоры мажут на мороси. Потолок = бинарное «был/не был», в лучшем случае грубая тройка «морось/умер./ливень» с большой ошибкой.
- **Роль датчиков в балансе:** бинарный **gate** для прогнозных осадков (засчитывать мм OM только при локальном подтверждении, обнулять при опровержении). Количество — из Open-Meteo. Измерять мм датчиками НЕ пытаться (самообман).
- **Честная альтернатива для количества:** дешёвый tipping-bucket дождемер (~30-40 у.е.) — единственное честное решение, если нужны реальные мм. Задание явно «без дождемера» → gate-логика, но стоит держать опцию в уме.

Источники: wet-bulb (memphisweather.blog); gust front (Stull Practical Meteorology Ch.15); fog/dew RH (ambientweather.com FAQ); ML rain classification (journalofbigdata.springeropen.com/articles/10.1186/s40537-021-00545-4); rain sensor lag (sciencedirect S135223100000323X); Ecowitt WS90 piezo undercount.

## 5. Что research диктует для дизайна (выводы)

1. **Баланс Dr по зонам, суточный пересчёт** — заменяет/дополняет мгновенный коэффициент. Чинит ночное занижение и скачки (баланс не зависит от времени суток).
2. **ET₀ Open-Meteo as-is × Kc(зона, сезон)**, single Kc, 3 сезонные константы. НЕ корректировать ET₀ локальным temp/hum.
3. **Прогноз vs факт разведены:** прогноз → «лить ли сегодня»; в Dr → только фактические past-day осадки Open-Meteo.
4. **Локальный gate осадков:** rain_sensor + ΔRH/ΔT подтверждают/опровергают прогнозные мм OM (бинарно). Количество из OM.
5. **Ночной rebalance** (раз/сут) по past-day факту OM — замыкание контура.
6. **Anti-drift:** clamp Dr∈[0,TAW]; reset Dr→0 по сильному дождю; старт Dr=0 после дождя; ручная корректировка + лог «прогноз vs факт».
7. **Флаг-переключатель** старый коэффициент ↔ новый баланс (откат на проде без деплоя).
8. **Параметры зон:** тип растения (газон/кусты/деревья/огород → Kc), тип почвы (→ TAW), Z_r, производительность спринклера (мм/ч, для перевода минут полива в мм I). Дефолты безопасные.
