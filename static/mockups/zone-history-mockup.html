Зафиксировано (по ответам):

  • Глобальная кнопка — только рядом с «Аварийная остановка» (не на /zones, /groups)
  • Период по умолчанию — 7 дней
  • «Все зоны» = одна линия (суммарные минуты), фильтр по группе вручную
  • Кнопка «Повторить» в списке запусков — не нужна
  • Литры если частично — показывать с пометкой «частично»
  • Сортировка — новые сверху
  • Колонка zone_runs.source — миграция (program / manual / api)

Библиотека графиков — рекомендация: Chart.js v4

  • Почему Chart.js: vanilla JS, multi-line + fill + dashed line из коробки, отличный tooltip-engine (можно положить разбивку «2 запуска: 10+15 мин»), responsive, MIT, ~60КБ gzip, огромное community.
  • Установка: один тег <script src="https://cdn.jsdelivr.net/npm/chart.js@4"> или локально через npm/static — выбор за тобой.
  • Альтернативы: ApexCharts (~140КБ, красивее, но тяжелее), uPlot (~40КБ, низкоуровневая), Plotly (~3МБ — слишком тяжёлая).
  • На какие фишки Chart.js завязываемся: type: 'line' + pointRadius для маркеров, два dataset'а (Факт / План) с borderDash для пунктира, кастомный tooltip-callback для разбивки по запускам, плагин annotation (опционально) для зелёной заливки «сэкономлено».

Новые уточнения ⚠

  • Сравнительный режим по умолчанию — галка включена сразу или пользователь сам включает? (в макете — включена для наглядности)
  • «План без алгоритма» = сколько было бы по расписанию (programs) без weather_skip / weather_adjustment? Или включая ручные запуски тоже? Я предполагаю первое — только запланированный автополив, ручные запуски всегда сверху факта.
  • Если расписания не было для зоны (например, новая зона) — что показывать на «План»? Скрывать линию или рисовать как 0?
