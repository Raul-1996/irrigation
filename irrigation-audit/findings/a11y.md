### Топ-3 риска для оператора в поле (пожилой пользователь, мобилка, солнце)

1. **Аварийная остановка / Emergency-stop кнопка** (`status.html:119`, CSS `status.css:243`) — одна кнопка `#d32f2f` на `#f5f5f5` body даёт контраст ~5.7:1 (AA OK, AAA FAIL). На солнце текст «Аварийная остановка» белый на красном даст ~4.2:1 (AA *FAIL* для обычного текста, для крупного OK). Шрифт `0.95rem ≈ 15.2px` — НЕ крупный текст по WCAG. **Это самая критичная функция продукта — должна быть AAA (7:1)**.
2. **Стартовая кнопка зоны / запуск полива** в bottom-sheet и run-popup строится на цвете `#00838f` без текстовой подсказки кроме эмодзи `▶` — на ярком солнце цвет «приглушённый бирюзовый» плохо различим, эмодзи рендерится мелко.
3. **Нет focus management в модалках:** ни в одной из 8 модалок (zones.html × 6, programs.html × 1, status.html × 2 sheet+popup, map.html photo-modal) нет focus trap и нет возврата фокуса. ESC обработчик отсутствует. Открытие модалки скринридером оставит пользователя в фоновом контенте.

### Путь (срочный → стратегический)

1. **24 ч (quick wins):** связать все `<label>` с `<input>` через `for`/`id` (mqtt.html, zones.html bulk-actions), добавить `alt` к декоративным изображениям/`alt=""`, `aria-label` к иконо-кнопкам.
2. **Sprint:** focus trap + ESC + восстановление фокуса для модалок; перейти с `<div onclick>` toggle-switch на `<button role="switch" aria-checked>`; ARIA live regions для toast/SSE.
3. **Архитектура:** усилить контраст палитры под полевой use-case (ввести «outdoor» CSS-токены), переделать heading hierarchy (один `<h1>` на страницу, h2 → h3 без скачков), добавить `<main>`-landmark обёртку и skip-link во всех страницах включая `login.html`/`404.html`.

## 1. WCAG 2.2 Findings (по success-criteria)

### 1.1.1 Non-text Content (Level A)
**Severity:** Serious. **WCAG-A FAIL.**

- `templates/status.html:193`, `templates/zones.html:336`: `<img id="photoModalImg" src="" alt="Фото зоны">` — **alt одинаковый для всех зон**, не описывает конкретную зону (полив каких растений). Должен формироваться динамически из имени зоны.
- `templates/map.html:83`: `img.alt = it.name` — лучше, но `it.name` — это имя файла, не описание содержимого карты. Для слепого пользователя «map_2024_v3.png» бесполезно.
- `templates/base.html:27`: `<span class="logo-icon">💧</span>` — эмодзи как иконка. Для AT не озвучивается смыслово (читается как «капля воды»). Парная `logo-text` рядом смягчает, но если экран узкий и `logo-text` скрыт — остаётся одна капля без alt.
- Десятки `<button>` с одной эмодзи: `▶ Запустить` (`status.html:143,238`), `🔍` (`status.html:144`), `+` (`programs.html:50` FAB), `✏️ Редактировать`, `📋 Дублировать`, `🗑 Удалить` (programs.html:206-209), `📤 Экспорт CSV`, `📥 Импорт CSV`, `➕ Добавить зону` (zones.html:31-33), `📍 По IP` (settings.html:76). Текст рядом частично решает, но `🔍` (`status.html:144`) и FAB `+` (`programs.html:50`) — **без текста и без `aria-label`** → screen reader скажет «кнопка плюс» / «кнопка лупа».

**Fix-направление:** все иконо-кнопки без видимого текста должны иметь `aria-label`; декоративные эмодзи — `aria-hidden="true"`; `<img alt>` — динамически из контекста.

### 2.1.1 Keyboard (Level A)
**Severity:** Critical. **WCAG-A FAIL.**

## 4. SSE / Live Updates

`services/sse_hub.py` пушит updates в браузер. На фронте обработчик не виден в шаблонах (вероятно в `status.js`, который не читался полностью). По шаблону:

- `notification-container` (base.html:66) — пустой. Нужно `<div ... role="region" aria-live="polite" aria-label="Уведомления">`.
- `connection-status` (status.html:12-14) — статичный HTML, JS только меняет `display: block/none`. Нужно `role="alert"`.
- `#sidebar-active-zone` (status.html:79) — отображает «Сейчас поливается». При смене зоны — silent для AT. Нужно `role="status" aria-live="polite"` либо отдельный live-region, обновляемый при ключевых событиях («Зона X запущена», «Полив завершён»).
- `#footer-time` (base.html:61) — обновляется каждую секунду. Если `aria-live` сюда добавить — будет спамить screen reader каждую секунду. **GOOD что live-region не нужен** — но пользователь должен иметь способ узнать время по запросу (text доступен по фокусу/поиску).

**Принцип:** обычные обновления (зона стартовала) → `aria-live="polite"`. Ошибки и emergency → `aria-live="assertive"` или `role="alert"`. Никогда не делать live на updating timer (footer-time).

## 8. Summary

### Финальный compliance level: **WCAG 2.2 — НЕ соответствует уровню A**

Минимум 5 критичных нарушений уровня A (label-input связь в mqtt.html, custom toggle без role/keyboard в programs.html, sortable headers без keyboard, dial drag-only без альтернативы клавиатурой, missing aria-label на иконо-кнопках) делают продукт юридически некорректным для базового A.

### Топ-3 проблемы для пользователя в поле

1. **Контраст secondary text `#999` повторяется 25+ раз и FAIL даже AA (2.85:1).**
   На солнце пожилой оператор не увидит подписи: zone-meta, weather-time, sheet-label, dial-unit и т.д. Главные значения (температура, время до конца полива) читаемы, а контекст вокруг них — нет. *Quick win:* глобальная замена `#999 → #595959` в `static/css/status.css`.

2. **Custom toggle-switch в programs.html — невозможно включить программу с клавиатуры.**
   `<div class="toggle-switch" onclick="toggleEnabled(...)">` (programs.html:174, 484-485) — нет `role="switch"`, `tabindex="0"`, `aria-checked`, keyboard handler. Программа полива не может быть включена/выключена screen reader-пользователем или оператором с моторными ограничениями. WCAG 2.1.1 A.

3. **Нет focus management ни в одной из 8 модалок + 2 sheets + 2 popups.**
   Открытие модалки не перемещает фокус внутрь, ESC не закрывает (в шаблоне обработчиков нет), фокус не возвращается на trigger. AT-пользователь после открытия модалки оказывается в фоновом контенте с никаким `aria-modal="true"` indication (атрибут есть в zones.html, но не enforced JS-ом). **Это блокирует завершение задачи** — пользователь буквально не может сохранить новую зону/программу/MQTT-сервер с AT.

### Путь (от срочного к стратегическому)

**День 1 — quick wins (см. §6):** 15 точечных фиксов — поднимают compliance с «не A» до «частично A». Время: ≤ 4 часа.
